# PruebaInventario
Esta prueba se basa en un MVP (Producto Mínimo Viable), por lo que no hay conexión con una base de datos; el inventario se gestiona mediante arreglos. Sin embargo, se cumple con los requisitos establecidos en el documento de instrucciones. El sistema tiene algunas limitaciones debido a esta implementación, como la imposibilidad de agregar productos con nombres duplicados.

No es necesario instalar dependencias o frameworks adicionales para ejecutar el proyecto, ya que está desarrollado únicamente con JavaScript, HTML y CSS. Se puede ejecutar localmente simplemente clonando el repositorio y abriendo el archivo 'index.html'.
